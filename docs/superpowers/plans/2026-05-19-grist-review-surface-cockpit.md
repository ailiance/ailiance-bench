# Grist Review Surface — cockpit-admin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "Review Datasets" route to the `cockpit-admin` app that embeds the Grist dataset-review doc, and vendor the review-console widget as a static asset so Grist can load it from `admin.ailiance.fr/review-console/`.

**Architecture:** `cockpit-admin` is a React 19 + TanStack Router (file-based) + Vite SPA in the `ailiance-demo` monorepo, served by nginx behind Keycloak SSO at `admin.ailiance.fr`. The widget HTML is dropped into Vite's `public/` dir so the build copies it to `dist/review-console/`; an nginx rule serves that path without the SPA fallback. A new file-based route renders an `<iframe>` pointing at the Grist doc, whose URL comes from a `VITE_GRIST_URL` build-time env var (unset → graceful fallback message).

**Tech Stack:** React 19, TanStack Router v1.85 (file-based) + Query v5, Vite, TypeScript strict, Tailwind 3.4, Vitest 2.1 + React Testing Library, Biome (lint), pnpm 9.

**Spec:** `ailiance-bench/docs/superpowers/specs/2026-05-19-grist-review-surface-design.md` (layers 1 + 3).

- [ ] **Step 3: Write the route component**

Create `apps/cockpit-admin/src/routes/review-datasets.index.tsx`:

```tsx
import { createFileRoute } from '@tanstack/react-router';

export function ReviewDatasetsPage() {
  const gristUrl = import.meta.env.VITE_GRIST_URL;

  if (!gristUrl) {
    return (
      <div className="p-6 text-sm text-gray-600">
        Grist review URL not configured. Set <code>VITE_GRIST_URL</code> at
        build time once the Grist review doc is published.
      </div>
    );
  }

  return (
    <iframe
      title="Grist dataset review"
      src={gristUrl}
      className="h-[calc(100vh-6rem)] w-full border-0"
    />
  );
}

export const Route = createFileRoute('/review-datasets/')({
  component: ReviewDatasetsPage,
});
```

## Task 5: Serve /review-console/ in nginx without the SPA fallback

**Files:**
- Modify: `deploy/nginx/spa.conf`

`deploy/nginx/spa.conf` serves the SPA from `root /usr/share/nginx/html;` with a catch-all `location / { try_files $uri $uri/ /index.html; }`. The vendored widget lands at `/usr/share/nginx/html/review-console/` (Vite copies `public/` into `dist/`, the Dockerfile copies `dist/` to the nginx root). The catch-all would still resolve `/review-console/` correctly *if* the files exist, but an explicit rule makes the intent clear and returns a real 404 instead of the SPA shell when a widget file is missing.

- [ ] **Step 1: Add the location block**

In `deploy/nginx/spa.conf`, immediately before the existing `location / {` block, add:

```nginx
location /review-console/ {
  try_files $uri $uri/ =404;
}
```

Leave the existing `location /assets/` and `location /` blocks unchanged.

- [ ] **Step 2: Verify nginx config syntax**

If Docker is available:
Run: `docker run --rm -v "$PWD/deploy/nginx/spa.conf:/etc/nginx/conf.d/spa.conf:ro" nginx:alpine nginx -t`
Expected: `syntax is ok` / `test is successful`.

If Docker is not available, visually confirm the new block is well-formed (balanced braces, semicolon-terminated directives) and sits before `location /`.

- [ ] **Step 3: Commit**


## Post-implementation (manual, not a code task)

2. Set `VITE_GRIST_URL` to that share URL in the cockpit-admin deployment build args (the Dockerfile `ARG` added in Task 2).
3. Rebuild and redeploy `cockpit-admin`.
4. In the Grist doc, set the custom-widget page's widget URL to `https://admin.ailiance.fr/review-console/index.html`.
5. Verify: visiting `admin.ailiance.fr/review-datasets` (behind Keycloak) shows the Grist doc, and the console-widget page loads the vendored widget.
