## Item 1 — Apertus 70B × emc-dsp-power

**Pre-requirements:**
- ☐ Confirm dataset: `ls ~/KIKI-Mac_tunner/data/eu-kiki-hf/emc-dsp-power/{train,valid}.jsonl`
- ☐ EuroLLM worker stopped (`launchctl stop ai.eurollm.worker` or `kill <pid>`)
- ☐ Free GPU mem ≥ 200 GB (`vm_stat | head -5`)

**Adapt the existing training script** (the per-domain wrappers hardcode one domain each). For `emc-dsp-power`:

```bash
ssh studio
cd ~/KIKI-Mac_tunner
# Edit DATA path in scripts/train_eu_kiki_apertus.py to:
#   DATA = str(PROJECT / "data" / "eu-kiki-hf" / "emc-dsp-power")
# Edit OUTPUT to:
#   OUTPUT = str(PROJECT / "output" / "eu-kiki" / "apertus-emc-dsp-power")
#
# Then launch:
.venv/bin/python scripts/train_eu_kiki_apertus.py 2>&1 | tee logs/apertus-emc-dsp-power.log
# OR use the batch script if it accepts a --domain flag:
.venv/bin/python scripts/train_eu_kiki_batch.py --model apertus --domain emc-dsp-power
```

Then copy the resulting adapter:

```bash
cp ~/KIKI-Mac_tunner/output/eu-kiki/apertus-emc-dsp-power/adapters.safetensors \
   ~/eu-kiki/output/adapters/apertus/emc-dsp-power/
# adapter_config.json will be auto-generated by the training run
```

**Expected output**: `~/eu-kiki/output/adapters/apertus/emc-dsp-power/adapters.safetensors` (~750 MB, same size as math-gsm8k).

## Sequencing recommendation

If you have one weekend window:

1. **Saturday morning** (gates 1-2): Items 1+2 (Apertus 70B back-to-back, ~7-8 h, while you're not using Studio).
2. **Saturday evening**: Item 5 Devstral stm32 (~2 h).
3. **Sunday**: Items 3+4 EuroLLM (need data prep first — could be a 2-3 h research task).
4. **Sunday afternoon**: Re-run Phase 3 with the 5 new adapters added (~3-4 h).

Total: ~15 h compute + ~3 h data prep + ~3 h Phase 3 rerun = ~21 h wall-clock spread over a weekend, or sequential weekdays.

## Anti-patterns

- ❌ Launch all 5 items in parallel: Studio M3 Ultra has 1 GPU, no parallelism for 70B+ training.
- ❌ Skip the EuroLLM worker stop: triggers OOM kernel panic during Apertus loading.
- ❌ Forget to copy the adapter to `~/eu-kiki/output/adapters/{model}/{domain}/`: the new training writes to `~/KIKI-Mac_tunner/output/eu-kiki/...`, not the eval-friendly path.
- ❌ Train without re-running `regen_adapter_configs.py`: new adapters will be missing `adapter_config.json` and Phase 3 will hit the same load_adapters error.
